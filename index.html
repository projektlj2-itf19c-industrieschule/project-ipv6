Was ist IPv6

IPv6 oder IPng (Internet Protocol Next Generation) genannt ist die Weiterentwicklung des aktuell Verbreiteten IPv4 Internet Protokolls.
Es ist in einem paketorientierten Netzwerk wie dem Internet für die Übertragung und Vermittlung von Datenpaketen zuständig.
IPv6 arbeitet auf Layer 3 des OSI-Modells und ist somit für die Paketvermittlung zuständig.

Ziel ist es, das bislang weit verbreitete IPv4 (Internet Protocol version 4) abzulösen und dessen Eigenschaften weiterzuentwickeln.
Im Wesentlichen erweitert das Protokoll den adressierbaren Bereich und löst damit das Problem der Adressknappheit.

Routing Präfix

Der Routing Präfix beschreibt den Wirkungsbereich der IP-Adresse und deren Funktion.
Diese wird vom Internet Service Provider vorgegeben und an den Endkunden weitergeleitet

Subnetz ID

Eine 16-Bit-Teilnetz-ID oder Subnetz ID.
Die Subnetz-ID beschreibt wie die private Netzwerktopologie aufgebaut ist bzw. wie das Netzwerk in Subnetze aufgeteilt wurde.
Sie stellt damit das Subnetz dar.

Interface ID/ Schnittstellen ID

64 Bit für die Interface ID/ Schnittstellen ID, die auch als Token bezeichnet wird.
Die Schnittstellen-ID wird entweder automatisch von der MAC-Adresse der Schnittstelle oder manuell im EUI-64-Format konfiguriert.

Vorteile von IPv6

Mehr nutzbare Adressen
  • Das ältere Internetprotokoll IPv4 hat die Möglichkeit rund 4,3 Milliarden verschiedene Adressen zu vergeben, IPv6 hingegen rund 340 Sextillionen.
    Mit IPv6 besteht die Möglichkeit jedem Gerät eine eigene statische Adresse zuzuweisen, die man von überall aus ansprechen könnte.
  • Eine IPv6-Adresse ist 96 bit länger als eine IPv4-Adresse und wird im Hexadezimal-Format formatiert:
    [42540594644626261931978030292209287468]10 Dezimal
    [2001:0540:0000:0000:0231:24FF:FE80:C12C]16 Hexadezimal
    [0010000000000001:0000010101000000:0000000000000000:0000000000000000:0000001000110001:0010010011111111:1111111010000000:1100000100101100]2 Binär
weitere Vorteile
  • Punkt-zu-Punkt-Verschlüsselung mit IPsec
  • Möglichkeit Geräte direkt aus dem Internet anzusprechen
  • Datenpakete bis 4 GByte (4.294.967.335 Byte) im Vergleich der Vorganger IPv4 kann nur Datenpakete bis zu 65,535 Kilobyte (65.535 Byte) übertragen
  • Multicast - Adressen, um eine Gruppe anzusprechen
  • Autokonfiguration der IPv6-Adressen möglich
  • die Headeroptionen der IPv6-Adressierung können die gleiche größe wie das Datenpaket selbst haben. Im Vergleich bei IPv4 sind die Headeroptionen maximal 40 byte groß
